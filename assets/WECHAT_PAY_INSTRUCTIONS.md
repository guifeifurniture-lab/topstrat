## 功能特性

### 1. 支付方式切换
- 每个价格卡片（入门锦囊/专业锦囊）都支持 PayPal 和微信支付两种方式
- 用户可通过按钮切换支付方式
- 界面简洁，与现有设计保持一致

### 2. 二维码展示
- 二维码图片路径：`/assets/wechat-pay-qr.png`
- 显示尺寸：280x280 像素
- 支持图片加载失败的降级显示

### 3. 支付流程
1. 用户选择"微信支付"选项
2. 扫描屏幕上显示的二维码
3. 在微信中确认支付金额（¥299 或 ¥799）
4. 点击"我已支付"按钮确认
5. 系统记录订单并发送通知

### 4. 交付流程
1. 用户扫码支付
2. 用户点击"我已支付"确认
3. 系统发送通知邮件给管理员
4. 管理员验证收款
5. 手动添加 GitHub Collaborator 权限
6. 发送确认邮件给用户

## 需要完成的步骤

### 1. 放置二维码图片（必须）
```bash
# 将微信支付二维码图片保存为 wechat-pay-qr.png
# 放置到以下目录：
website/assets/wechat-pay-qr.png
```

**获取二维码方法：**
1. 打开微信 - 我 - 服务 - 收付款 - 二维码收款
2. 设置收款金额为固定金额（可选）
3. 保存收款二维码图片
4. 重命名为 `wechat-pay-qr.png`
5. 放置到 `website/assets/` 目录

### 2. 配置后端 API（可选）
如需自动处理支付确认，需要：
1. 安装 Node.js 依赖：`npm install express nodemailer`
2. 配置邮件服务
3. 在服务器中引入 API 路由

### 3. 测试支付流程
1. 打开 `vault.html` 页面
2. 点击任意价格卡片的"微信支付"按钮
3. 确认二维码正常显示
4. 测试"我已支付"按钮功能

   - 管理员需手动验证每笔收款
   - 核对金额和付款时间
   - 保留支付凭证记录

3. **用户隐私**
   - 保护用户邮箱地址
   - 不存储敏感支付信息
